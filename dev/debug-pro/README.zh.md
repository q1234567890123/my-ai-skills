## 这个技能是干什么的

这个技能用于专业调试、BUG 定位、崩溃排查、回归问题、测试失败、构建失败、性能异常、集成问题。

核心目标：不要猜，不要乱改，用证据一步步缩小范围，修根因，不修表象。

## 调试原则

调试是证据管理。每一个判断都要能被日志、堆栈、测试、输入、状态、提交记录或运行环境支持。

## 调试流程

1. 复现问题。
2. 记录准确症状：错误信息、堆栈、输入、环境、设备、时间、分支。
3. 明确预期行为和实际行为。
4. 看最近变更和依赖边界。
5. 建立假设树。
6. 先验证最高概率且成本最低的假设。
7. 修根因，不只消除报错。
8. 能加回归测试就加。
9. 验证失败场景和相邻场景。

## 常见根因

- 空值或未初始化状态。
- 异步顺序错误。
- 重复事件监听。
- 生命周期假设错误。
- 缓存过期。
- 坐标空间不一致。
- 序列化/Schema 不匹配。
- 时区和语言环境问题。
- 依赖版本变化。
- 错误被过早吞掉。

## 修复规则

- 修复位置要靠近不变量的所有者。
- 信任边界要加校验。
- 异步流程要处理取消和过期结果。
- 事件、定时器、tween、订阅要清理。
- 不要用大范围 `try/catch` 掩盖问题。
- 没证据不要重写系统。

## 报告模板

AI 使用本技能时，应输出：

- 症状；
- 预期；
- 复现方式；
- 证据；
- 根因；
- 修复；
- 验证；
- 剩余风险。
